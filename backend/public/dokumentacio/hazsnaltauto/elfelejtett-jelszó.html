Elfelejtett jelszó

Erre a funkcióra a laravel beépített funkcióját használtam.

E-mail küldés

Habár valóban sokkal egyszerűbb lett volna simán a php mailto függvénnyel egy ideiglenes jelszót küldeni, ekkor még nem tudtam hogy ez is elfogadható megoldás...

A Laravel megoldás úgy működik mint a legtöbb weboldal jelszó-visszaállítása: Küld egy linket a megadott e-mail címre amely megnyitásával a felhasználó új jelszót állíthat be.

Ehhez a következők szükségesek:

  • Egy view, amely bekéri az e-mail címet

  • Egy másik view, ahol az új jelszót tudjuk beállítani

AuthController.php

$request->validate(['email' => 'required|email']); $user = User::where('email', $request->email)->first(); if (!$user) { \Log::error('User not found with email: ' . $request->email); } $status = Password::sendResetLink( $request->only('email') ); return $status === Password::RESET_LINK_SENT ? back()->with(['status' => __($status)]) : back()->withErrors(['email' => __($status)]);

Jelszó beaállítás

Miután a felhasználó megynitott a linket, egy új jelszót állíthat be.

Mivel bejelentkezési hiba miatt nem tudtunk SMTP szolgáltatást, vagy más biztonságos levelezési protokolt használni, ezért nagy eséllyel a felhasználó spam mappájába kerülnek az email-ek.

NewPasswordController.php

$request->validate([ 'token' => 'required', 'email' => 'required|email', 'password' => ['required', 'confirmed', Rules\Password::defaults()], ]); $status = Password::reset( $request->only('email', 'password', 'password_confirmation', 'token'), function ($user, $password) { $user->forceFill([ 'password' => Hash::make($password), ])->save(); $user->setRememberToken(Str::random(60)); } ); return $status === Password::PASSWORD_RESET ? redirect()->route('login')->with('status', __($status)) : back()->withErrors(['email' => [__($status)]]); }
